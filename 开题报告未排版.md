# 基于大模型的学科竞赛智能管理系统开发

## 一、选题的目的和意义

当前，人工智能在软件开发中的应用正逐步深化，AI 辅助开发方法成为提升软件工程效率和质量的重要趋势。AI 技术早已今非昔比，与传统开发方式相比，AI 在代码自动生成、优化和测试方面展现出巨大潜力。如何将 AI 技术有效融入软件开发流程，提升开发效率和代码质量，是当前软件工程领域的研究热点。本项目旨在研究 AI 如何在软件工程实践中提升开发效率、优化代码质量，并减少人工干预，而获奖管理系统则是该方法论的实验载体。

当前许多高校的获奖管理系统较为简陋，存在多个痛点：数据录入、审核和查询流程高度依赖人工操作，缺乏智能化数据处理；管理系统往往无法与咨询系统联动，导致信息孤岛问题严重；此外，系统升级和功能扩展缓慢，难以适应高校日益增长的管理需求。因此，迫切需要一套更为智能化、自动化的获奖管理系统。本项目选择该应用场景，不仅是为了解决实际痛点，更是为 AI 辅助开发方法提供一个理想的实践平台。

在方法论研究方面，本项目希望突破传统的人工开发模式，采用大模型驱动的自动化开发与测试流程，以提升软件工程的智能化水平。通过集成 Cursor 作为开发工具，并结合 Dify 和 LangGraph 的 Agent 工作流，我们将探索基于 AI 的微单元开发和测试，使代码的编写、优化和调试过程更加高效智能。此外，本项目还将采用 Vue.js 作为前端框架，结合 FastAPI 构建高效的后端系统，以实现前后端分离、响应式界面设计和高性能 API 服务。

从研究价值来看，本项目不仅关注 AI 辅助开发方法的探索，还希望通过实际应用场景验证其有效性。近年来，代码自动生成、智能补全、自动化测试等 AI 技术逐渐成熟，但如何将这些技术有效融入完整的软件开发流程，仍然是一个值得深入研究的问题。大语言模型 (LLM) 的应用也不止是诸如 ChatGPT、DeepSeek Chat 这类的对话系统，在由于代码生成时，我们需要一个更加专业的结构——即 Agent。Agent 是一个可以在代码中进行操作的实体，它可以在代码中进行操作，如添加、删除、修改代码等。本项目将通过完整的实践过程，评估 AI 驱动的软件开发模式在真实应用中的可行性，并为未来更大规模的软件工程变革提供参考。

在实际应用方面，本系统将为高校教务管理人员、教师和学生提供更加高效、智能的竞赛信息管理平台。系统不仅支持获奖信息的录入、审核和查询，还能利用图像识别技术自动识别获奖证书，提高数据录入效率，并支持与高校其他教务管理平台的对接，确保数据一致性和可拓展性。

总体而言，本项目的研究价值体现在两个方面：一是通过 AI 辅助开发方法提升软件开发的智能化水平，提高代码质量和开发效率；二是通过 AI 驱动的学科竞赛管理系统优化高校的竞赛管理流程，为 AI 在软件开发中的实践应用提供宝贵经验。

接下来的内容以及后续的工作都将围绕两部分展开：一是 AI 辅助开发工具的设计和实现，二是基于 AI 的学科竞赛管理系统的开发实践。这将贯穿整个毕业设计过程。

### 2. 开题前的准备工作

#### 2.1 竞赛管理系统需求分析及技术选型

学科竞赛依托于专业知识，将理论与实践相结合，是培养创新人才的关键途径。它对于促进学校人才培养模式的创新、实践教学的改革，激发学生的学习热情、拓宽知识视野，以及培育学生的创新意识、实践能力和团队协作精神都具有不可忽视的作用。近年来，随着国家对大学生学科竞赛的日益重视，各类由政府、学术机构、企业以及高校等组织的竞赛活动日益增多，大学生的参与规模也在不断扩大。为了引导师生参与竞赛，学校建立了不同级别的竞赛库。学院管理人员承担着统计、汇总数据以及填写上报各类表格的任务。如何提高管理与服务质量，并且加强赛事与专业学科发展的紧密结合，已经成为高等教育机构越来越关注的问题。

北京工商大学高度重视学生参与各类学科竞赛和科技竞赛，并已将竞赛项目扩展至 113 项。鉴于学生参与竞赛的数量不断增加，传统的手工管理方式已不再适应当前的需求，因此迫切需要开发一套信息化系统来有效管理竞赛项目的获奖信息。在设计学科竞赛获奖录入系统之前，我们需对学校的竞赛情况进行综合分析，明确系统的功能需求和性能需求。结合多年的竞赛管理经验，从二级学院的角度出发，我们期望系统能够实现以下功能：

1. 学号工号信息补充：因全校重名的学生和较多，必须填写学号和工号确保信息准确。

1. 建立学校竞赛项目库：系统应包含一个全面的竞赛项目库，用于存储和管理所有竞赛的详细信息。

1. 自动化获奖信息提取：系统应能够通过拍照等方式自动提取获奖信息，包括但不限于竞赛名称、参赛学生等关键信息。

1. 管理员审核：竞赛库中标注的都是竞赛的最高等级，例如 A 竞赛最高级别是国家级，那么它的子竞赛和选拔赛就分别是省部级和校级，这就必须有学院管理员审核认定。

1. 学分授予和教师工作量统计：根据学生的竞赛成果和所获奖项，系统应自动计算并授予相应的创新实践学分；同时，系统还应能够统计教师的竞赛指导工作量。

通过实现这些功能，学科竞赛获奖录入系统将大大提高竞赛管理的效率和准确性，为学校的教学和科研工作提供有力支持。

根据需求分析，我们需要设计一款基于 B/S 架构的（即基于浏览器的）高校学科竞赛信息管理系统。我们设计申报端用于学生和教师登录进行竞赛成果申报，同时设计管理端用作学院教务管理员等相关负责人员对申报结果的审核统计和导出。系统以前中后三台形式组织，同时前台申报端和后台管理端分别具有其服务前后端，而相关的文件与数据储存等公共服务交由中台处理，前后台不保存关键数据。

针对以上需求，我们可以使用前后端分离模式进行开发，前端采用基于 Vite 脚手架的 Vue 单页应用，后端采用 Python FastAPI，数据库采用 MySQL，并通过 Tortoise ORM 进行数据操作。前端和后端通过 RESTful API 进行通信，实现前后端分离。

Vue.js 是一套用于构建用户界面的渐进式框架，它的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。更重要的是，Vue.js 提供了单文件组件 (SFC) 的开发方式，从页面上的一个小组件到整个页面，都可以用一个 .vue 文件来表示，这对于人工智能辅助开发工具的开发非常有利——我们可以很方便的将整个开发流程拆分成多个小组件，然后通过 AI 辅助开发工具逐个生成代码片段、最终组成完整的页面系统。

FastAPI 是一个现代的 Python Web 框架，它基于 Python 3.6+ 的标准类型提示，使用 Pydantic 进行数据验证，支持异步请求处理，性能优异。FastAPI 的主要特点是快速、易用、少代码、自动文档生成等，非常适合用于构建高性能的 Web 服务。同样的，它的函数式编程风格和异步处理特性，也非常适合与 AI 辅助开发工具结合，实现代码的自动生成和优化。

Tortoise ORM 是一个异步 ORM 框架，它支持异步请求处理，非常适合用于 FastAPI 后端服务的数据操作。利用 Tortoise ORM，我们可以将复杂的数据库操作封装成简单的异步函数，无需关心数据库连接和事务管理，无需直接操作 SQL 语句，大大提高了开发效率。

#### 2.2 辅助开发工具的技术选型

参考前文中的大语言模型对开发工作的介入方式，我们可以将我们目前需要用到的 AI 辅助开发工具的功能粗略的归为两类：

协作式介入 (Collaborative)：即 AI 与开发者共同完成代码编写、优化和调试等工作。这类工具通常以插件或 IDE 的形式存在，可以与开发者的编辑器无缝集成，提供代码补全、错误检查、代码优化等功能，例如 GitHub Copilot、Cursor 等。

代理式介入 (Agent)：即 AI 作为一个独立的实体，模拟功能模块开发、测试等工作。这类工具通常以工作流的形式存在，可以模拟开发团队的操作，生成代码片段、测试用例等，例如 Dify、LangGraph 等。

在本项目中，我们将主要关注代理式介入的 AI 辅助开发工具，即 Agent。我们首先需要设计一个 Agent 工作流，用于模拟开发团队的操作，生成代码片段、测试用例等。这里我们同时采用图形化的 Agent 设计工具 Dify 和更具自由度的代码式 Agent 设计工具 LangGraph，以满足不同场景下的需求。同时，Cursor 和 Copilot 等协作式介入工具也将作为辅助工具，协助部分需要人工干预的工作。
